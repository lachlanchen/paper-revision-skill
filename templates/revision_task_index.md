# Revision Task Index

Use this index when the number of plan files becomes large.

| Date | Plan file | Task origin | Scope | Status | Commit | Notes |
|---|---|---|---|---|---|---|
| <YYYY-MM-DD> | `references/<plan>.md` | Reviewer <n>, Comment <m> | <manuscript/response/figure/global> | <draft/approved/executed/verified/committed> | `<hash>` | <short note> |

## Status Definitions

- `draft`: plan exists but has not been approved.
- `approved`: plan has been reviewed and can be executed.
- `executed`: source files have been modified.
- `verified`: build and PDF checks passed.
- `committed`: changes were committed to git.
