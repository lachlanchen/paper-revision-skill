# Response Letter and Proof Workflow

This reference covers reviewer response letters, minor-revision letters, and proof-correction notes.

## Contents

1. Response letter structure
2. Location strategy
3. Figure and equation references
4. Proof correction format
5. Funding and acknowledgement corrections
6. Tone rules

## 1. Response Letter Structure

For each reviewer comment:

1. Quote or summarize the comment accurately.
2. Thank the reviewer briefly.
3. State the change made.
4. Explain why the change addresses the concern.
5. Give the current manuscript or Supplementary Material location.

Avoid turning the response into a long changelog. The reviewer needs to see what changed and where to check it.

## 2. Location Strategy

Use current compiled PDFs for final locations.

Preferred location formats:

- `main manuscript, page 2, paragraph beginning "..."`
- `Supplementary Material, page S3, paragraph beginning "..."`
- `main manuscript, Fig. 3 caption`
- `Supplementary Table S1`

Use redlined PDFs to find what changed, but do not rely on redline page numbers if redline layout differs from the final manuscript.

When the journal template has no line numbers, page and paragraph anchors are more stable than guessed line references.

## 3. Figure And Equation References

Keep namespaces clear:

- Main manuscript figures: `Fig. 1`, `Fig. 2`, etc.
- Supplementary figures: `Fig. S1`, `Fig. S2`, etc.
- Response-letter-only figures: call them response figures or otherwise distinguish them from manuscript figures.

Before finalizing:

- Search every figure label and ensure each main figure is cited.
- Check equation numbering after adding equations.
- If the response letter says an equation changed, verify whether the current paper actually changed that equation or added a new one.

## 4. Proof Correction Format

When using a journal proof system, choose `Request Revisions` if any correction remains.

Use this item format:

```text
1. Please <requested correction>.
Location: <page, column, paragraph or section>.
Current proof text:
`<current text>`
Corrected text:
`<corrected text>`
```

For example:

```text
Please restore the missing citation to Fig. 2 in the main text.
Location: page 2, paragraph beginning "We bin event timestamps..."
Current proof text:
`... without external timing signals.`
Corrected text:
`... without external timing signals (Fig. 2).`
```

## 5. Funding And Acknowledgement Corrections

Funding corrections should use the full funder name requested by the journal or production team.

Use compact funding metadata when possible:

```text
Funding. <Funder full name> (<program or grant number>).
```

If the journal asks for acknowledgement-style funding wording, provide a full sentence:

```text
The work described in this paper was partially supported by <funder full name> (Project No. <grant number>).
```

Acknowledgement changes after acceptance may be treated as content changes. Phrase them as:

```text
If possible, please update the Acknowledgment section to add <name>.
```

## 6. Tone Rules

Use direct, polite production language:

- `Please restore...`
- `Please update...`
- `Please add...`

This is polite enough for proof corrections. Avoid over-softening the request when production needs an exact edit.

For reviewer responses, avoid wording that sounds defensive. Prefer:

- `Thank you for pointing this out.`
- `We have clarified...`
- `We now state...`
- `This change appears in...`

Avoid unsupported praise, exaggerated claims, and wording that implies the reviewer made an obvious mistake.
